<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client (product-form) -> Server Action (saveProduct) | Input do revendedor cruza aqui; validação client é conveniência, a fronteira real é a revalidação Zod + parseBRLPrice no servidor |
| Server Action -> Postgres (RLS) | saveProduct escreve escopado por store_id resolvido de auth.uid() via getOwnedStore |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-05 | Tampering | campo preço (parseFloat cru trunca 199,90 -> 199; input não-numérico) | medium | mitigate | parser dedicado parseBRLPrice (rejeita null/negativo) + revalidação productSchema no servidor (Task 2, Pitfall 3) |
| T-03-06 | Elevation of Privilege | saveProduct sem owner-scoping gravaria em loja alheia | high | mitigate | getOwnedStore() resolve store_id de auth.uid() antes de qualquer insert; RLS de products (Plan 03-01) é a rede final |
| T-03-07 | Input Validation | campos obrigatórios/enum burlados via chamada direta ao Server Action | medium | mitigate | productSchema.safeParse revalidado no servidor; brand="Outra" exige brand_other; nunca confiar só no client |
</threat_model>
