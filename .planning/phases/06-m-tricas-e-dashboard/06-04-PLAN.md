<objective>
Criar o shell de navegação compartilhado do painel (D-05/D-06/D-07): a `AdminSidebar` (sidebar fixa no desktop via `<aside hidden md:flex>`, hambúrguer + drawer `<dialog>` nativo no mobile via `<button md:hidden>` + `.showModal()`), e um grupo de rotas aninhado `(admin)/(painel)/` com seu próprio `layout.tsx` que envolve só as páginas autenticadas. Mover `dashboard/`, `produtos/` e `configuracoes/` para dentro de `(painel)/`, trocar a raiz `<main>`→`<div>` de cada página movida (o `<main>` único passa a viver no layout), e repontar o teste de dark mode.

Purpose: substitui a navegação ad-hoc (botões soltos no dashboard) por navegação estruturada, sem vazar controles autenticados ("Sair da conta") para as telas públicas de login/cadastro. É a fatia de escopo ampliado explicitamente pedida pelo usuário (D-05). A correção idiomática do Next.js App Router é estrutural (grupo de rotas aninhado, invisível na URL), não lógica de runtime — nenhuma URL muda.

Output: painel autenticado com sidebar/drawer responsivo; páginas públicas de auth intocadas; suíte verde.

Roda por último (Wave 3) porque MOVE a `dashboard/page.tsx` reescrita em 06-03 — depende de 06-03 estar concluído.
</objective>
