<objective>
A fatia crítica e inegociável do produto: o cliente final abre a página de detalhe, seleciona um tamanho disponível e dispara a mensagem de pedido pronta no WhatsApp do revendedor — a única conversão que importa. Monta o Server Component totalmente dinâmico da rota `/loja/[slug]/[produto]` (mesma disciplina "sem cache" da vitrine, estoque sempre fresco), o Client Component `ProductOrderPanel` (pílulas de tamanho com guard de mouse+teclado, "Pedir agora" com `<a href>` real e `preventDefault` condicional, "Copiar mensagem" sempre visível), e a Server Action `logOrderClick` fire-and-forget que registra o clique sem nunca atrasar a abertura do WhatsApp.

Reúne o padrão mais load-bearing da fase (o `<a>` sempre-clicável com `preventDefault` condicional — NÃO é o anti-padrão `window.open`, é a navegação nativa do anchor quando válido). Como não há jsdom no projeto, a verificação real é a matriz manual obrigatória de dispositivos/navegadores — bloqueador de encerramento da fase pelo próprio ROADMAP.

Purpose: Entregar PED-01/02/03/04 ponta a ponta e a captura fire-and-forget para a Fase 6.
Output: rota de detalhe funcional + CTA de WhatsApp confiável + fallback de cópia + registro de clique.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| cliente anônimo → Server Action (logOrderClick) → Postgres (RLS) | Server Action é a borda; a RLS (WITH CHECK de 05-01) é a aplicação real da segurança, não o código |
| dados do produto/template → string da mensagem → href do anchor | conteúdo do revendedor entra no href; risco é encoding (resolvido em 05-02), não injeção de HTML |
| navegação nativa do `<a href>` → app WhatsApp | confiabilidade em webview in-app depende de anchor real (nunca window.open) — validada na matriz manual |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-09 | Tampering | Insert de order_clicks com par forjado via Server Action | high | mitigate | WITH CHECK no banco (05-01) rejeita par inconsistente/não-publicado independente dos argumentos; insert BARE sem .select() |
| T-05-10 | Denial of Service | Fire-and-forget que trava/atrasa a conversão (WhatsApp não abre) | high | mitigate | startTransition com resultado ignorado; navegação nativa do anchor nunca espera a action (D-10); target="_blank" reduz cancelamento por page-unload |
| T-05-11 | Spoofing/Reliability | window.open programático falhando em webview in-app do Instagram/WhatsApp | high | mitigate | Sempre `<a href>` real com preventDefault condicional (nunca window.open/router.push); confirmação obrigatória na matriz manual |
| T-05-12 | Tampering | XSS via nome/descrição do produto na DOM ou no link | low | accept | Escape padrão de JSX do React; nenhum dangerouslySetInnerHTML no projeto |
