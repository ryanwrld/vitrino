# Phase 5 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — manual Tailwind v4 (CSS-first, no `tailwind.config.js`), no component library initialized (`components.json` absent) |
| Preset | not applicable |
| Component library | none — every interactive element (buttons, pills, inputs) is hand-rolled with `<button>`/`<a>`/`<input>` + Tailwind utility classes, identical to the pattern already used in `product-filters.tsx`, `size-grid.tsx`, `load-more-button.tsx` |
| Icon library | `lucide-react` 1.24.0 (already installed, legitimacy-audited in Phase 2 — see STATE.md "lucide-react aprovado no gate de legitimidade (T-02-SC)") |
| Font | System stack via `globals.css` (`font-family: Arial, Helvetica, sans-serif` on `body`) — no new font added this phase |

**Note on shadcn gate:** This project reached Phase 5 of 6 without ever initializing shadcn (`components.json` does not exist). Four prior phases (1–4) built a complete, internally consistent hand-rolled Tailwind design language (documented below, extracted directly from `product-card.tsx`, `product-filters.tsx`, `size-grid.tsx`, `store-hero.tsx`, `image-with-fallback.tsx`, `load-more-button.tsx`). Introducing shadcn now — mid-milestone, one phase before completion — would fragment the design system rather than strengthen it. **Decision: continue with the existing hand-rolled convention, `Tool: none`.** Registry Safety section below is therefore not applicable.

## Spacing Scale

Declared values (must be multiples of 4) — matches the scale already in use project-wide:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps (e.g. availability dot + label), `gap-1` |
| sm | 8px | Compact stacks — card internal gaps, size-pill grid gaps (`gap-2`) |
| md | 12px–16px | Default element spacing — panel section gaps (`gap-3`/`gap-4`), button horizontal padding (`px-4`) |
| lg | 24px | Section padding — hero/panel vertical breathing room |
| xl | 32px | Layout gaps between major page sections (photo gallery → info block → order panel) |

Exceptions:
- **44px (`min-h-11`/`min-w-11`) touch targets** — mandatory for every size pill, the "Pedir agora" CTA, and "Copiar mensagem" CTA. This is an established project-wide exception (already used in `size-grid.tsx`, `product-filters.tsx`'s search input) required for reliable mobile tap accuracy on this phase's single most important interaction.
- **6px vertical padding (`py-1.5`) on filter chips** — pre-existing exception from Phase 4, not introduced here; not used by any new component in this phase.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | Regular (400) | 1.5 (`leading-normal`) — size-pill numbers, message preview text, back-link label |
| Label | 14px (`text-sm`) | Medium (500) | 1.5 (`leading-normal`) — "Pedir agora"/"Copiar mensagem" button text, price, brand/line secondary text |
| Heading | 20px (`text-xl`) | Medium (500) | 1.2 (`leading-tight`) — "Escolha o tamanho" section label, mirrors `size-grid.tsx`'s existing `text-xl font-medium` |
| Display | 24px (`text-2xl`) | Bold (700) | 1.2 (`leading-tight`) — product name (`<h1>`) at the top of the detail page, mirrors `store-hero.tsx`'s existing `text-2xl font-bold` |

Exception: **12px (`text-xs`)** is used project-wide for tertiary meta text (e.g. `secondaryLine` in `product-card.tsx`) and applies here to the photo-gallery position indicator ("Foto 1 de 3") and the tooltip bubble text ("Selecione um tamanho") — kept small and dismissible so it never competes visually with the CTA.

Weight discipline: only **2 weights used in this phase's new components** — Regular (400, default body) and Medium (500, labels/buttons/pills). Bold (700) is reserved exclusively for the product name display heading, mirroring the one existing precedent (`store-hero.tsx` h1) rather than introducing a new use.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **"Pedir agora"** — locked by ROADMAP/CONTEXT.md D-02/D-03, never conditionally relabeled |
| Secondary CTA | **"Copiar mensagem"** — locked by D-07/D-08, always visible beside/below the primary CTA, never a conditional fallback |
| Guard tooltip (PED-04) | **"Selecione um tamanho"** — locked verbatim by ROADMAP Success Criteria #4 and REQUIREMENTS PED-04; appears with the shake animation, auto-dismisses after ~2.5s or on next valid interaction |
| Copy-success toast | **"Mensagem copiada!"** — locked verbatim by CONTEXT.md D-07 |
| Copy-failure toast | **"Não foi possível copiar. Tente novamente."** — not locked upstream; proposed here matching the existing error-toast tone used in `load-more-button.tsx` (`toast.error(result.error)`) |
| Sold-out product state | **"Esgotado"** badge (muted dot + `#6B6B6B` text) — reuses the exact existing pattern from `product-card.tsx`; no new empty-state copy needed since the size grid itself communicates unavailability per-pill (all pills struck through) |
| Error state (product not found / unpublished / hidden by sold-out rule) | New minimal `not-found.tsx` for this route, in Portuguese (matching the rest of the app — no other page in this codebase renders English copy): Heading **"Produto não encontrado"**, Body **"Este produto não está mais disponível ou o link mudou."**, Link **"Voltar para a loja"** → `/loja/[slug]`. Rationale: Pitfall 8 in RESEARCH.md flags direct-link bypass of the sold-out-hide rule as a real, expected path a customer will hit via a stale shared link — the default Next.js English 404 would be the *only* English text in the entire product, so a small Portuguese `not-found.tsx` scoped to this route segment is in-scope for this phase, not a separate deliverable. |
| Destructive confirmation | N/A — this phase has no destructive actions (no delete/cancel flow; clicking a sold-out pill silently no-ops per RESEARCH.md Pattern 2, it does not require a confirmation dialog) |

## Component Specifications

*(Supplementary detail beyond the template's minimum — included because this phase's interaction edge cases are the primary risk per RESEARCH.md.)*

### Size pill (public, on `/loja/[slug]/[produto]`)
Visually and structurally identical to the existing admin `size-grid.tsx` pill (`min-h-11 min-w-11`, `rounded-lg`, `text-base`), with one difference: public pills are strictly **2-state** (available/sold-out), never 3-state (no "not included" state exists on the public side — only sizes actually in `product_sizes` are rendered at all).

| State | Classes (Tailwind) |
|-------|---------------------|
| Available, unselected | `border-[#F5F5F3] bg-white text-[#111111]` |
| Available, selected | `border-[#00C46A] bg-[#00C46A] text-white` |
| Sold out | `pointer-events-none text-[#6B6B6B] line-through opacity-60 border-[#F5F5F3] bg-[#F5F5F3]`, `tabIndex={-1}` |

### "Pedir agora" CTA
- Always a real `<a href>` (never `disabled`), per D-02. `href="#"` when no size selected, real `wa.me` URL once selected — see RESEARCH.md Pattern 3.
- Default: `min-h-11 w-full rounded-lg bg-[#00C46A] px-4 py-2 text-center text-sm font-medium text-white transition` (mirrors `load-more-button.tsx`'s existing CTA button styling, adapted to an anchor).
- Invalid-click state: apply a `shake` CSS class keyed by an incrementing `shakeKey` (forces remount so rapid repeated clicks re-trigger the animation — RESEARCH.md Pitfall 4). Shake motion: small horizontal translate oscillation, ~400ms, no color change.
- Never shows a toast/confirmation on successful click (D-03) — the WhatsApp app opening is the only confirmation.

### "Copiar mensagem" CTA
- `min-h-11 w-full rounded-lg border border-[#6B6B6B] px-4 py-2 text-center text-sm font-medium text-[#6B6B6B] transition` — deliberately neutral/secondary styling (never `#00C46A`) to keep visual hierarchy subordinate to "Pedir agora".
- Icon: `lucide-react`'s `Copy` (or `ClipboardCopy`) at `h-4 w-4`, left of the label, `gap-1`.
- Always visible (D-08) — never conditionally rendered based on wa.me link failure detection.

### Product photo gallery
- Reuses `ImageWithFallback` verbatim (same `#F5F5F3` background + `ImageOff` icon fallback already proven in Phase 4).
- Swipeable/scrollable horizontal strip on mobile, `aspect-square`, `rounded-xl`, matching the card's existing image treatment.
- Position indicator ("Foto 1 de 3") at `text-xs text-[#6B6B6B]`, centered below the gallery — only rendered when the product has more than 1 photo.

### Back navigation
- A small `ChevronLeft` (lucide-react) + "Voltar" text link at the top of the page, `text-sm text-[#6B6B6B]`, navigating to `/loja/[slug]` (preserving no filter state — acceptable simplification, filters are not part of this phase's scope).
