<objective>
Build the backend that makes real-time slug uniqueness possible without breaking tenant isolation. The `stores` RLS policy (`owner_id = auth.uid()`, from 0001) structurally blocks any RLS-bound client from seeing OTHER tenants' rows, so a naive `.from('stores').select().eq('slug', x)` always reports "available" even when a slug is taken (02-RESEARCH.md Pitfall 1 — verified against the codebase). The fix is a `SECURITY DEFINER` Postgres RPC `is_slug_available` returning ONLY a boolean, granted only to `authenticated`, with a pinned `search_path`. Then create the three settings Server Actions (`checkSlugAvailability`, `updateStoreSlug`, `saveStoreSettings`) — all in one file since they share `src/lib/settings/actions.ts` — with the mandatory `[BLOCKING]` schema push + type regen ordered between the migration file and the integration tests.

Purpose: satisfy D-03 (real-time uniqueness) correctly and provide the save actions the UI slices call.
Output: the migration, the pushed RPC, regenerated DB types, three Server Actions, and three integration tests against the real Supabase project.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated revendedor → Server Action → Postgres | Slug availability crosses tenant isolation via an elevated-privilege RPC |
| Server Action → stores.slug write | Two revendedores may race for the same slug (TOCTOU) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-03 | Information Disclosure | is_slug_available RPC (SECURITY DEFINER bypasses RLS) | high | mitigate | Function returns ONLY a boolean via `not exists`, never row columns; granted only to `authenticated`; regression test asserts no cross-tenant row data is reachable |
| T-02-04 | Elevation of Privilege | SECURITY DEFINER search_path hijack | high | mitigate | Pin `set search_path = public, pg_temp` on the function definition (Task 1) |
| T-02-05 | Tampering | slug uniqueness TOCTOU race between debounce check and save | medium | mitigate | DB-level UNIQUE constraint on stores.slug (from 0001) is the true enforcement; updateStoreSlug catches error code 23505 and returns the friendly message (Task 3, tested) |
| T-02-06 | Information Disclosure | authenticated slug enumeration via the boolean RPC | low | accept | Slugs are public vitrine URLs by design (meant to be shared); existence of a slug leaks nothing sensitive; grant limited to authenticated keeps it off the anonymous surface |
</threat_model>
