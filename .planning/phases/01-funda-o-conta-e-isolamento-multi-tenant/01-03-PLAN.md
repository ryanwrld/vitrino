<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| formulário → Server Action | Entrada não confiável revalidada com Zod no servidor |
| navegador → layout (admin) | getUser() revalida a sessão a cada request de rota protegida |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-07 | Spoofing | (admin)/layout.tsx | high | mitigate | Gate usa `getUser()` (revalida contra o servidor Supabase), nunca `getSession()` — grep confirma |
| T-01-08 | Information Disclosure | signInAction / mensagens de erro | medium | mitigate | Mensagem genérica "Email ou senha inválidos"; sem confirmar/negar existência de conta |
| T-01-09 | Repudiation | renovação de sessão silenciosa | medium | mitigate | SessionWatcher avisa em SIGNED_OUT (D-03); Server Actions retornam erro de auth explícito (mitiga Assumption A2) |
| T-01-10 | Elevation of Privilege | acesso ao dashboard sem onboarding | low | mitigate | Guard de dados `requireCompletedOnboarding` separado do gate de auth |
| T-01-SC | Tampering | (sem novos installs neste plano) | low | accept | Nenhuma dependência nova é instalada neste plano |
</threat_model>
