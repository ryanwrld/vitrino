<objective>
Entregar a fatia de gerenciamento: o revendedor edita um produto existente (formulário pré-preenchido com campos, tamanhos e fotos), exclui um produto (removendo linhas E arquivos do storage, atrás de confirmação), e publica/despublica (D-10 — rascunho vira publicado por ação explícita). Cada ação dá toast imediato.

Purpose: sem editar/excluir/publicar, o catálogo é write-only e nunca fica limpo. Publicar é o portão que a vitrine pública (Fase 4) consome via status='published' — decisão de schema do Plan 03-01, ação aqui. A limpeza de storage no delete evita arquivos órfãos (Pitfall 1 — cascade do Postgres NÃO apaga blobs). Exclusão é hard delete com diálogo nativo de confirmação (mesmo padrão do slug-editor da Fase 2).
Output: edição, exclusão com cleanup, e publish/draft toggle, todos com feedback visual.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Server Action (update/delete/publish) | productId vindo do cliente; o servidor sempre re-resolve owner via getOwnedStore e confia na RLS, nunca no id sozinho |
| Server Action -> Storage + Postgres | deleteProduct remove blobs do bucket owner-scoped e linhas via cascade |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-11 | Tampering / Information Disclosure | editar/excluir/publicar produto de outra loja passando productId alheio | high | mitigate | getOwnedStore() + RLS de products (subquery owner) faz UPDATE/DELETE afetar 0 linhas cross-tenant; testado (Task 1) |
| T-03-04 | Denial of Service (custo/quota) | arquivos órfãos no bucket após exclusão (cascade não apaga blobs) | medium | mitigate | deleteProduct busca storage_path e chama storage.remove ANTES do cascade (Pitfall 1, Task 2) |
| T-03-12 | Tampering | despublicar/publicar em massa via chamada direta | low | accept | ação reversível e de baixo impacto; owner-scoped por getOwnedStore/RLS; sem gate adicional |
</threat_model>
