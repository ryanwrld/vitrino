<objective>
Corrigir o gap major de contraste (UAT M-4 + achado bônus) reportado no onboarding e presente em TODAS as páginas admin: quando o SO do usuário está em dark mode, o bloco de media query de esquema escuro herdado do boilerplate do create-next-app inverte o fundo para quase-preto, enquanto os labels usam cores hex fixas escuras (`text-[#111111]`, `text-[#0D3D2B]`) — resultando em texto quase invisível. O projeto NÃO tem dark mode (CLAUDE.md confirma fora de escopo).

Purpose: Restaurar legibilidade em todo o painel admin (cadastro, login, dashboard, esqueci-senha, redefinir-senha, onboarding) para qualquer revendedor cujo dispositivo tem preferência de dark mode ativa — um default extremamente comum em macOS/iOS/Windows.

Output: `globals.css` forçando esquema claro (sem flip de dark mode) + `bg-white` explícito em todo `<main>` do admin + teste de regressão estático.

Deferido (nota de backlog, NÃO fazer neste plano): a migração de longo prazo dos valores hex hardcoded (`text-[#111111]` etc.) para tokens `@theme` conectados às variáveis CSS. A diagnose (`.planning/debug/onboarding-invisible-labels.md`) marcou isso como opcional/menor prioridade; com o flip de dark mode neutralizado o bug reportado fica 100% resolvido, e o refactor cosmético amplo em 6 páginas adiciona risco de regressão visual sem benefício funcional. Fica registrado como item de backlog para uma futura fase de polimento de UI.
