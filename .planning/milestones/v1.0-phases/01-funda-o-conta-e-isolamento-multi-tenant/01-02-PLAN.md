<objective>
Criar o schema multi-tenant fundacional (`stores`, `store_settings`) com RLS habilitado na mesma migration, constraint UNIQUE no slug, coluna explícita `onboarding_completed_at`, e o bucket de Storage `store-assets`; rodar o [BLOCKING] push do schema para o Postgres do Supabase; e provar o isolamento entre tenants com duas contas reais seedadas.

Purpose: RLS é a ÚNICA fronteira de autorização do projeto (nunca reimplementar isolamento em código de aplicação — Padrão 4). Um esquecimento aqui é vazamento de dados silencioso (Armadilha 4) ou colisão de slug silenciosa (Armadilha 3). Este schema sustenta LOJA-01 (nome/logo/cor/frase), WPP-01 (whatsapp_e164) e WPP-02 (message_template), preenchidos no onboarding (Plan 05).
Output: migration aplicada ao banco, tipos gerados, e teste de isolamento verde com duas contas.
</objective>
